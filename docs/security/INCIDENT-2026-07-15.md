# Repository secret exposure incident — 2026-07-15

## Classification

Repository-history release blocker. Live WordPress containment and authentication-material rotation were completed on 2026-07-15; Git history remediation and repository-visibility containment remain open.

## Confirmed exposure

A generated WordPress backup directory was committed to a public repository. The tracked material included non-empty WordPress authentication keys/salts and an installer log containing operational server metadata. Database password fields in the inspected configuration copy were empty; database credential exposure must nevertheless be checked across the complete Git history before closure.

No secret values, SSH identifiers, internal account names or full server paths are reproduced in this document.

## Repository active-tree containment completed

- Removed the exposed configuration copy from the active tree.
- Removed the generated installer log from the active tree.
- Added explicit ignore rules for WordPress configuration, Duplicator artifacts, backups, database exports, archives, private keys, logs and installer remnants.
- Removed files that were tracked while also matching `.gitignore`, including WordPress core/runtime/vendor artifacts and generated QA screenshots.
- Added a CI security gate that rejects tracked ignored files, forbidden operational paths and common secret signatures.
- Security Gate, CSS Gate and PHP Lint passed on the containment change before merge.

## Live production remediation completed

A controlled SSH operation was completed on 2026-07-15. The execution produced `SECURITY_ROTATION_PASS` and did not deploy content or purge cache.

Verified outcomes:

- `siteurl` and `home` remained `https://nuvanx.com`.
- The static front page remained page ID 9.
- The active theme remained `nuvanx-medical`.
- `wp-config.php` passed PHP syntax validation.
- WordPress bootstrap/core validation passed.
- All eight standard WordPress authentication keys/salts changed relative to the protected pre-rotation backup.
- `WP_CACHE_KEY_SALT` is present with a rotated value.
- Active sessions were destroyed for the three administrator accounts.
- The broad WPVibe remote-editing plugin is inactive.
- Duplicator backups and installer remnants are absent from the public web root and retained in protected quarantine outside it.
- Production returned HTTP 200 after remediation.
- No site content was deployed.
- No cache purge was performed.
- Temporary remediation scripts were removed.

A protected pre-change configuration backup and a quarantine directory exist outside the public web root with restrictive permissions. Their full physical paths are intentionally not committed to the public repository.

An initial shell pass encountered SiteGround incompatibility with process substitution. A second pass used a compatible loop and completed all quarantine and verification checks. The first-pass changes were not reverted because credential rotation, session invalidation and the initial quarantine move had already completed successfully; the second pass verified the final state.
